# 世界坐标系：
世界坐标系基于右手定则，大拇指X，食指Y、中指Z，国际惯例用RGB三色表示。

# 相机坐标系：
一般用3个字母命名，如RUB、LUF、RDF，依次表示相机坐标系XYZ三个轴的方向。容易混淆的是B和F，这里是相对于相机朝向来描述的，沿着相机镜头向拍摄物体的方向是F，反之则是B。

## 常见相机坐标系：
RUB: Meshlab、NeRF、OpenGL、Blender
RDF: OpenCV、Colmap
LUF: Pytorch3D、Mitsuba
DRB: LLFF

### Pytorch3D：
- 使用行主序（row-major）矩阵，通常在使用（numpy、colmap、opengl）时采用列主序（column-major）矩阵。因此，在设置相机旋转矩阵 R 时，请务必将其转换为行主序矩阵。
- 在 PyTorch3D 中指定相机位姿 RT 做渲染时，使用的是 world2camera（w2c）矩阵。

#### Pytorch3D定义相机的方式
look_at_view_transform(distance, elevation, azimuth)

这个函数设置的相机看向世界坐标系原点，
azimuth是沿着z轴正向向x轴正向旋转的角度
elevation是沿着x轴正向向y轴正向旋转的角度

## 相机参数
​计算机视觉：通常将 ​外参矩阵​ 定义为从世界坐标系到相机坐标系的变换矩阵。
​机器人学：通常将 ​Pose 矩阵​ 定义为从相机坐标系到世界坐标系的变换矩阵。
外参矩阵和 Pose 矩阵是互为逆矩阵的关系

### pose矩阵的物理含义
其中，旋转矩阵 R 是一个 3×3 的正交矩阵，描述了相机的方向（即相机坐标系相对于世界坐标系的旋转）。平移向量 t 是一个 3×1 的向量，描述了相机的位置（即相机坐标系原点在世界坐标系中的坐标）。


#### 旋转矩阵 R 的物理含义
旋转矩阵 R 的每一列分别对应相机局部坐标系的三个基向量在世界坐标系中的表示。具体来说：

​第一列：相机局部坐标系的 ​右向向量（Right）​​ 在世界坐标系中的表示。
​第二列：相机局部坐标系的 ​上向向量（Up）​​ 在世界坐标系中的表示。
​第三列：相机局部坐标系的 ​前向向量（Forward）​​ 在世界坐标系中的表示。
这些向量是相机局部坐标系的基向量，它们定义了相机的方向。

​右向向量（Right）​：
物理意义：相机局部坐标系的 X 轴方向。
在世界坐标系中的表示：R 的第一列 r=(r11 ,r21,r31)。
​上向向量（Up）​：
物理意义：相机局部坐标系的 Y 轴方向。
在世界坐标系中的表示：R 的第二列 u=(r12,r22,r32)。
​前向向量（Forward）​：
物理意义：相机局部坐标系的 Z 轴方向。
在世界坐标系中的表示：R 的第三列 f=(r13,r23,r33)。

## 相机坐标系的转换

从一个坐标系到另一个坐标系，由于坐标轴的定义不同，通常需要做处理。比如从Colmap到Pytorch3D。
```python
# 已知colmap下的相机外参矩阵w2c
# 1.获取相机的pose矩阵，即c2w矩阵
c2w = torch.inverse(w2c)
# 获取相机朝向的旋转矩阵R和相机位置的平移向量t
R, t = c2w[:3, :3], c2w[:3, 3:]
# 在pose矩阵上完成两个相机坐标系的变换，from RDF to LUF for Rotation
R = torch.stack([-R[:, 0], -R[:, 1], R[:, 2]], 1) 
# 获取新坐标系下的pose矩阵和外参矩阵
new_c2w = torch.cat([R, t], 1)
w2c = torch.linalg.inv(torch.cat((new_c2w, torch.Tensor([[0,0,0,1]])), 0))
# 变成行主序 
R, T = w2c[:3, :3].permute(1, 0), w2c[:3, 3] # convert R to row-major matrix
R = R[None] # batch 1 for rendering
T = T[None] # batch 1 for rendering
```


## 从一个坐标系的相机矩阵到另一个坐标系的相机矩阵，举例：colmap到pytorch3d
这里的相机矩阵默认是w2c矩阵，也就是外参矩阵中的旋转矩阵




```python
# 获取了colmap的外参矩阵4x4 w2c

# w2c 到 c2w，也就是相机的pose矩阵 
c2w = torch.inverse(w2c) # to c2w
# 分离旋转矩阵和平移向量
# 这里的T是相机在世界坐标系中的坐标
# 这里的R
R, T = c2w[:3, :3], c2w[:3, 3:]
# 在pose矩阵上做坐标系的变换，from RDF to LUF for Rotation
R = torch.stack([-R[:, 0], -R[:, 1], R[:, 2]], 1) 
# 获取新坐标系下的pose矩阵和外参矩阵
new_c2w = torch.cat([R, T], 1)
w2c = torch.linalg.inv(torch.cat((new_c2w, torch.Tensor([[0,0,0,1]])), 0))
# 变成行主序 
R, T = w2c[:3, :3].permute(1, 0), w2c[:3, 3] # convert R to row-major matrix
R = R[None] # batch 1 for rendering
T = T[None] # batch 1 for rendering
```

## 自定义设置相机外参

Example：
现有一个ply格式的点云，需要设置虚拟相机对其进行渲染，将该点云放入meshlab中进行观测。meshlab是标准的rub格式，然后pytorch3d是luf格式，所以需要对相机的第一个和第三个分量取负

```python3
# 首先在meshlab中获取两个向量，一个是Forward，这个就是看向哪里，哪里就是forward
# 这里在meshlab中看，选择的是（1，0，0）
# 然后看，如果想拍摄角争取的话， up的向量沿着z轴正向，也就是（0,0,1）
# 接下来有了Foward和up之后，可以直接用np.cross求解这个right向量，如果想正确的话，应该是z 叉乘x得到y
# np.cross是基于右手定则的，然后meshlab、pytorch3d的相机坐标系也都是右手定则的

# 由于pytorch3d就是luf，这里直接构造一个luf的pose矩阵
forward = np.array([1,0,0])
up = np.array([0,0,1])
left = np.cross(up, forward)
# 然后单位化，得到left
left = left / np.linalg.norm(left)
R = np.column_stack((left, up, forward))
R = torch.tensor(R, dtype=torch.float32)



T = torch.tensor([0, 0., 0], dtype=torch.float32)
C2W = torch.eye(4)
C2W[:3, :3] = R
C2W[:3, 3] = T
C2W = C2W.unsqueeze(0).repeat(cfg.n_views, 1, 1).to(cfg.device)
w2c = torch.linalg.inv(C2W)
R = w2c[:, :3, :3].permute(0, 2, 1)
T = w2c[:, :3, 3]
```
